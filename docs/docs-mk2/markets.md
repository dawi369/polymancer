## Implementation Notes

### How to Fill This Template

1. Visit https://polymarket.com/explore
2. Filter by category and sort by volume
3. Click into each market to get:
   - Condition ID (from URL: `/event/[slug]-[condition_id]`)
   - Exact question text
   - Slug
   - Tags
4. Verify liquidity is consistently >$100k
5. Ensure market doesn't resolve within 30 days

### Database Seeding

Once filled, export this data to seed `market_cache` table:

```sql
-- Seed script (run once)
INSERT INTO market_cache (market_id, question, slug, tags, outcome_prices, volume_24h, fetched_at)
VALUES 
  ('[condition_id]', '[question]', '[slug]', ARRAY['[tag1]', '[tag2]'], '{"yes": 0.65, "no": 0.35}', 150000, now()),
  ...
```

### Market Updates

- Markets should be refreshed weekly to remove resolved/closed markets
- Add new markets to replace resolved ones
- Keep total at exactly 50 for consistent AI context size
- Document date of last refresh

### Category Distribution Strategy

Current distribution optimized for:
- **Politics/Crypto (30 markets)**: Highest volume, most active
- **Sports (10 markets)**: Seasonal variety
- **Science/Tech/Pop Culture (10 markets)**: Engagement and diversity

Adjust distribution based on user feedback and actual trading patterns.
